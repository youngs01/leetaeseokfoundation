# 문의 확인 여부 기능 구현 완료

## 개요
관리자가 들어온 문의를 확인(읽음) 여부로 구분하고 관리할 수 있는 기능이 추가되었습니다.

## 구현 내용

### 1. 데이터베이스 변경
- **테이블**: `inquiries`
- **새로운 컬럼**: `is_read` (boolean, 기본값: false)

### 2. 타입 정의 업데이트
- `inquiry-store.ts`의 `Inquiry` 타입에 `isRead: boolean` 필드 추가

### 3. API 엔드포인트

#### GET /api/inquiries
- 모든 문의 조회 (admin_auth 필수)
- isRead 필드 포함

#### PATCH /api/inquiries/[id]
- 특정 문의의 읽음 상태 변경
- 요청 본문: `{ "isRead": boolean }`

### 4. 관리자 페이지 기능

#### 대시보드 (admin/page.tsx)
- **미확인 문의 카드**: 빨간색으로 강조
- 미확인 문의 개수를 별도로 표시

#### 문의 관리 (admin/inquiries/page.tsx)
- **필터 탭** (4가지):
  1. **미확인** (빨간색 - 새로 추가): 아직 확인하지 않은 문의만 표시
  2. **전체** (파란색): 모든 문의 표시
  3. **강의 문의** (파란색): 강의 관련 문의만
  4. **일반 문의** (파란색): 일반 문의만

- **상태 표시**: 각 문의 첫 번째 컬럼
  - ✓ (초록색): 확인된 문의
  - ⚠️ (빨간색): 미확인 문의 (클릭만으로 상태 변경 가능)

- **행 색상**:
  - 미확인 문의: 파란색 배경 (눈에 띄도록)
  - 확인 문의: 흰색 배경

### 5. 인터페이스 개선
- 미확인 문의를 한눈에 볼 수 있도록 상단에 배치
- 클릭만으로 빠르게 상태 변경 가능
- 대시보드에서 미확인 문의 개수를 빨간색으로 표시

## 사용 방법

### 더 알아보기
1. 관리자 로그인 후 대시보드로 이동
2. "미확인 문의" 카드에서 미확인 문의 개수 확인
3. "문의 관리" 메뉴 클릭
4. "미확인" 탭 클릭하면 미확인 문의만 필터링
5. 각 문의의 상태 아이콘 클릭으로 확인/미확인 토글
